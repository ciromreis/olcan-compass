## Executive Summary

This document provides a comprehensive analysis of the current v2.5 state after Codex's AI-first shell implementation, identifies critical gaps in commerce/CMS integration, proposes a redesigned Aura/companion UX architecture, and lays out a detailed execution plan for a less-capable agentic AI to complete the consolidation.

### Current State Assessment

**✅ Completed by Codex:**
- 3-column AI-first shell (sidebar + main + AuraRail)
- Journey spine with ActionItem/JourneySnapshot model
- Unified commerce/auth consolidation (single base URL, token namespace)
- Bridge-backed checkout flow
- Decommissioned legacy direct-Medusa product route
- Fixed broken checkout retry path
- Pruned primary nav and sitemap exposure
- Backend pytest scoping

**✅ Validation:**
- `pnpm --filter @olcan/web-v2.5 test`: 38/38 passing
- TypeScript compilation: clean
- Build: passing with eslint/type-checking enabled
- pytest: scoped to `tests/` (async tests skipped due to missing pytest-asyncio in live env)

**❌ Critical Gaps Identified:**

1. **Commerce/CMS Integration:**
   - MedusaJS client exists but not fully woven into core flows
   - PayloadCMS integration is superficial (only fetching chronicles)
   - Redundant product data structures (ecommerceStore vs medusa-client)
   - No true product catalog sync between backend commerce API and frontend
   - Missing CMS-driven content recommendations in journey spine

2. **Aura/Companion UX:**
   - AuraRail is passive and disconnected from user actions
   - No conversational interface or AI-driven guidance
   - Gamification feels bolted-on, not integrated
   - Presence phenotype logic exists but UI doesn't leverage it
   - Missing contextual product/service recommendations in Aura flow

3. **Legacy Code:**
   - Old route surfaces still exist (not deleted, just hidden from nav)
   - Duplicate stores: `companionStore.ts`, `canonicalCompanionStore.ts`, `realCompanionStore.ts`
   - Multiple marketplace stores with unclear boundaries
   - `documents/` route exists alongside `/forge` (redundant)
   - `/companion` route exists alongside `/aura` (redundant)

4. **Backend-Frontend Mismatches:**
   - API endpoints in `api.ts` don't match actual backend routes
   - Commerce API uses `/commerce/public/products` but frontend expects `/marketplace/products`
   - CMS URL hardcoded to localhost:3001 (no production fallback)
   - Missing error boundaries for API failures

5. **Route/Task Management:**
   - Routes feel rigid (no dynamic milestone reordering)
   - Task management scattered across sprints/applications/forge
   - No unified "What do I need to do?" view
   - Deadlines/milestones not surfaced prominently enough

6. **AI Era UX Deficiencies:**
   - No chat interface for Aura companion
   - No voice interaction for interviews
   - No AI-driven content suggestions in Forge
   - No contextual help/tooltips powered by AI
   - Missing "Ask Aura" floating button

### Problems

1. **MedusaJS is orphaned:** Client exists, but no pages use it. Products come from DEMO_PRODUCTS or `/commerce/public/products` API.
2. **PayloadCMS is underutilized:** Only fetches chronicles for content recommendations. No product metadata, no archetype overrides in production.
3. **Duplicate product sources:** `ecommerceStore` has DEMO_PRODUCTS, `api-client` fetches from `/commerce/public/products`, `medusa-client` would fetch from MedusaJS (unused).
4. **No single source of truth:** Commerce data scattered across multiple stores and API clients.

### Key Changes

1. **Single Commerce Store:** Merge `ecommerceStore`, `medusa-client`, and commerce API client into one unified service.
2. **CMS Integration:** PayloadCMS becomes the source of truth for product metadata, editorial content, and archetype overrides.
3. **Backend Proxy Layer:** FastAPI proxies MedusaJS and PayloadCMS, adding caching, auth, and business logic.
4. **Unified Journey Store:** Merge routes, sprints, and applications into a single `journeyStore` with a unified task/deadline view.
5. **Aura as Conversational UI:** Floating chat button + rail, powered by Vertex AI Gemini, context-aware recommendations.

#### 1.1 Backend Commerce Proxy

**File:** `apps/api-core-v2.5/app/routers/commerce_proxy.py` (NEW)

```python
from fastapi import APIRouter, Depends, HTTPException
from app.core.medusa_client import medusa_client
from app.core.cms_client import cms_client
from app.core.cache import cache_service
from app.models.user import User
from app.core.auth import get_current_user

router = APIRouter(prefix="/api/commerce", tags=["commerce"])

@router.get("/products")
async def list_products(
    category: str | None = None,
    search: str | None = None,
    limit: int = 50,
    offset: int = 0,
    current_user: User | None = Depends(get_current_user),
):
    """
    Unified product listing from MedusaJS + PayloadCMS metadata.
    Cached for 5 minutes.
    """
    cache_key = f"products:{category}:{search}:{limit}:{offset}"
    cached = await cache_service.get(cache_key)
    if cached:
        return cached

    # Fetch from MedusaJS
    medusa_products = await medusa_client.list_products(
        category_id=category,
        q=search,
        limit=limit,
        offset=offset,
    )

    # Enrich with CMS metadata
    product_ids = [p["id"] for p in medusa_products["products"]]
    cms_metadata = await cms_client.get_product_metadata(product_ids)

    # Merge
    enriched = []
    for product in medusa_products["products"]:
        meta = cms_metadata.get(product["id"], {})
        enriched.append({
            **product,
            "is_olcan_official": meta.get("is_olcan_official", False),
            "is_featured": meta.get("is_featured", False),
            "journey_tags": meta.get("journey_tags", []),
            "recommended_for": meta.get("recommended_for", []),
        })

    result = {"items": enriched, "count": medusa_products["count"]}
    await cache_service.set(cache_key, result, ttl=300)
    return result
```

#### 1.2 Frontend Commerce Service

**File:** `apps/app-compass-v2.5/src/services/commerce.ts` (NEW)

```typescript
import { resolveApiBaseUrl } from "@/lib/api";

export interface Product {
  id: string;
  title: string;
  handle: string;
  description: string;
  price: number;
  currency: string;
  thumbnail: string | null;
  images: string[];
  category: string;
  tags: string[];
  is_olcan_official: boolean;
  is_featured: boolean;
  journey_tags: string[];
  recommended_for: string[];
}

class CommerceService {
  private baseUrl: string;

  constructor() {
    this.baseUrl = resolveApiBaseUrl();
  }

  async listProducts(params?: {
    category?: string;
    search?: string;
    limit?: number;
    offset?: number;
  }): Promise<{ items: Product[]; count: number }> {
    const query = new URLSearchParams();
    if (params?.category) query.set("category", params.category);
    if (params?.search) query.set("search", params.search);
    if (params?.limit) query.set("limit", params.limit.toString());
    if (params?.offset) query.set("offset", params.offset.toString());

    const response = await fetch(`${this.baseUrl}/commerce/products?${query}`, {
      headers: { "Content-Type": "application/json" },
    });

    if (!response.ok) {
      throw new Error(`Failed to fetch products: ${response.statusText}`);
    }

    return response.json();
  }

  async getProduct(handle: string): Promise<Product> {
    const response = await fetch(`${this.baseUrl}/commerce/products/${handle}`);
    if (!response.ok) {
      throw new Error(`Failed to fetch product: ${response.statusText}`);
    }
    const data = await response.json();
    return data.item;
  }
}

export const commerceService = new CommerceService();
```

#### 1.3 Consolidate Commerce Store

**File:** `apps/app-compass-v2.5/src/stores/commerce.ts` (REPLACE `ecommerceStore.ts`)

```typescript
import { create } from "zustand";
import { devtools, persist } from "zustand/middleware";
import { commerceService, type Product } from "@/services/commerce";

interface CommerceState {
  products: Product[];
  featuredProducts: Product[];
  currentProduct: Product | null;
  isLoading: boolean;
  error: string | null;

  fetchProducts: (params?: any) => Promise<void>;
  fetchFeaturedProducts: () => Promise<void>;
  fetchProduct: (handle: string) => Promise<void>;
  clearError: () => void;
}

export const useCommerceStore = create<CommerceState>()(
  devtools(
    persist(
      (set) => ({
        products: [],
        featuredProducts: [],
        currentProduct: null,
        isLoading: false,
        error: null,

        fetchProducts: async (params) => {
          set({ isLoading: true, error: null });
          try {
            const { items } = await commerceService.listProducts(params);
            set({ products: items, isLoading: false });
          } catch (error) {
            set({
              isLoading: false,
              error: error instanceof Error ? error.message : "Failed to fetch products",
            });
          }
        },

        fetchFeaturedProducts: async () => {
          set({ isLoading: true, error: null });
          try {
            const { items } = await commerceService.listProducts({ limit: 10 });
            const featured = items.filter((p) => p.is_featured);
            set({ featuredProducts: featured, isLoading: false });
          } catch (error) {
            set({
              isLoading: false,
              error: error instanceof Error ? error.message : "Failed to fetch featured products",
            });
          }
        },

        fetchProduct: async (handle) => {
          set({ isLoading: true, error: null });
          try {
            const product = await commerceService.getProduct(handle);
            set({ currentProduct: product, isLoading: false });
          } catch (error) {
            set({
              isLoading: false,
              error: error instanceof Error ? error.message : "Failed to fetch product",
            });
          }
        },

        clearError: () => set({ error: null }),
      }),
      { name: "commerce-store" }
    ),
    { name: "commerce-store" }
  )
);
```

```python
from fastapi import APIRouter, Depends, HTTPException
from app.models.user import User
from app.core.auth import get_current_user
from app.services.vertex_ai import vertex_ai_service
from app.services.journey import journey_service

router = APIRouter(prefix="/api/aura", tags=["aura-ai"])

@router.post("/chat")
async def aura_chat(
    message: str,
    current_user: User = Depends(get_current_user),
):
    """
    Send a message to Aura AI companion.
    Returns AI-generated response with contextual recommendations.
    """
    # Get user's journey context
    journey_context = await journey_service.get_snapshot(current_user.id)

    # Build prompt
    prompt = f"""
    You are Aura, the AI companion for {current_user.full_name} on their international mobility journey.

    Journey Context:
    - Route Readiness: {journey_context["metrics"]["routeReadiness"]}%
    - Lifecycle Stage: {journey_context["lifecycle"]}
    - Next Best Action: {journey_context["nextBestAction"]["title"] if journey_context["nextBestAction"] else "None"}

    User Message: {message}

    Respond in a friendly, supportive tone. Provide actionable advice.
    If relevant, recommend products or services from the marketplace.
    """

    # Call Vertex AI Gemini
    response = await vertex_ai_service.generate_text(prompt)

    return {"message": response, "context": journey_context}
```
