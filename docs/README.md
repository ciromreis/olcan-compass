## Estado atual

- O núcleo comercial da v2.5 é `diagnóstico -> rotas -> forge -> entrevistas`.
- Marketplace entra como camada premium de destravamento humano.
- A camada hoje chamada de `companion` deve funcionar como leitura personalizada e retenção, não como tema dominante do produto.
- Parte relevante da documentação legada continua útil como referência, mas não deve ser tratada como fonte única de verdade.

## Regra prática

Ao desenvolver código novo:

1. valide primeiro o alinhamento com `PRODUCT_CORE_CONSOLIDATION.md`
2. consulte a estratégia integrada e o PRD
3. só depois navegue por documentos satélites

## Observação

Se um documento antigo disser que o produto está “100% completo”, “production ready” ou centralizado em guildas, batalhas ou estúdio, trate isso como histórico desatualizado.
